Netty 3.8.0.Final released

by normanmaurer
on 08-Nov-2013

Even if most work these days is related to Netty 4 we have not forgotten all our 3.x users. So I'm happy to announce the release of Netty 3.8.0.Final. This is mostly a bug-fix release but ships some changes in the SPDY codec which deserve the version bump. Please refer to the next section for more details. The release closes 15 issues / tasks.

Most important changes / fixes

  • WebSocketClientHandshaker.handshakeComplete must be volatile (#1945)
  • Fix regression in HashedWheelTimer when handling timeouts that overflow the thick (#1994) and ((#1959)
  • Websocket: an exception could be raised if the client sends message after it thinks the handshake is complete (#1933)
  • Deadlock for SSL connections when closing during an exception handler (#1817)
  • SPDY: remove SPDY/2 support (#1919)

Visit here for the complete list of the changes.

As always please let us know if you find any issues. We love feedback!

Special notes

There is something in this release that deserve more details. We removed SPDY/2 support as it is dropped from clients.

Acknowledge

Every idea and bug-report counts and so we thought it is worth mentioning those who helped in this area. You deserve it!
