Netty 4.0.12.Final released

by normanmaurer
on 08-Nov-2013

After some very busy days, I'm happy to announce the release of Netty 4.0.12.Final. This is mostly a bug-fix release but also include some small performance improvements. Please refer to the next section for more details. The release closes 15 issues / tasks . As expected from a bug-fix release it is API compatible to previous version out of the 4.0.x tree.

Release often, release early as our users deserve it!

Most important changes / fixes

  • SslHandler renegotiation not working (#1964)
  • Fix regression in HashedWheelTimer when handling timeouts that overflow the thick (#1994) and ((#1959)
  • After the channel closes, client not always receives last part of data. (#1952)
  • DuplicatedByteBuf not correctly expand capacity (#1802)
  • Internal buffer capacity increase shouldn't discard read bytes (#1800)

Visit here for the complete list of the changes.

As always please let us know if you find any issues. We love feedback!

Special notes

There is something in this release that deserve more details.

Two of our users notified us that while they where working on implement APN with Netty they noticed that sometimes they was not able to receive all data from the Socket before it was closed.

After investigate the culprint was found... In Netty we close the Socket automatically when a IOException was thrown during writing the data to the underlying Socket. Because of this all pending data on it was lost, as there was not way for the user to fetch it. This is most of the times not a problem as usually there is nothing left to read and it is not possible to read anything from the Socket after such an IOException was thrown.

Anyway in some situations there maybe something left that you can read. To not break behavior we added a new configration that allows to disable or enable the automaticaly close of the Socket when a write throws an IOException.

channel.config().setAutoClose(false);

This is true by default to keep the old behaviour. So you may want to change it to false.
